AutoSys Job Scheduling

Enterprise job scheduling and automation

1 What is AutoSys?
AutoSys is an enterprise job scheduling tool used to automate and manage batch jobs. It handles job dependencies, triggers, calendars, and notifications. Widely used in Banking, Finance, and ETL environments.
2 What is a Box Job?
A Box job groups related jobs together for easier control. It acts like a container and runs only when conditions inside are satisfied. Stopping or starting a Box affects all jobs inside it.
3 What is a Command Job?
A command job runs shell scripts, commands, or programs. It is the most commonly used job type and is executed on the machine specified by the "machine" attribute.
4 What is JIL?
JIL (Job Information Language) is used to create, modify, or delete jobs. It defines job properties like command, owner, start time, and conditions without using a GUI.
5 How do you create a job using JIL?
Create a .jil file with job attributes like insert_job, job_type, and command. Load it using jil < file.jil.
6 How do you force start a job?
Use the command: sendevent -E FORCE_STARTJOB -J jobname. It runs the job immediately, ignoring conditions. Used during testing or emergencies.
7 What are the different job statuses?
Common statuses include: STARTING, RUNNING, SUCCESS, FAILURE, TERMINATED, ON_HOLD, and ON_ICE.
8 Difference between ON_HOLD and ON_ICE?
ON_HOLD: Job does not run and does NOT process events; dependencies stay active.
ON_ICE: Job does not run but passes "SUCCESS" to downstream jobs to avoid blocking dependencies.
9 How do you check job details?
autorep -j jobname -q for detailed info.
autorep -j jobname -r to check run history.
10 What happens if a job fails?
The job goes to FAILURE status. Notifications are triggered based on configuration. Support teams analyze logs, fix the issue, and restart using sendevent.
11 What is sendevent?
sendevent is used to send commands to AutoSys jobs (e.g., STARTJOB, FORCE_STARTJOB, JOB_ON_ICE). Essential for command line control.
12 What is a File Watcher Job?
Monitors the existence or modification of a file. Runs once the file appears or changes. Common in ETL workflows.
13 What is a Calendar?
Defines specific run dates (weekdays, monthly, holidays). Jobs linked to a calendar run only on those designated days.
14 How do you restart a job?
Use sendevent -E STARTJOB -J jobname. Use FORCE_STARTJOB to ignore conditions. Always check logs before restarting.
15 What is Max Run Alarm?
Alerts when a job exceeds its normal runtime. Used to detect hung jobs. Configured with the max_run_alarm attribute.
16 Advantages of Control-M?
Control-M is used to schedule, monitor, and manage batch jobs from a single interface. Works across Windows, UNIX, Linux, and cloud platforms.

UNIX Commands

Essential commands for daily operations

1 What is the ls command?
Lists files and directories. Options like -l (long format), -a (hidden files), and -h (human readable) provide details.
2 How does grep work?
Searches for patterns inside files or outputs. Supports regex. Example: searching logs for errors.
3 What does ps do?
Displays running processes. ps -ef provides full details. Often combined with grep to find specific processes.
4 Difference between kill and kill -9?
kill sends SIGTERM (graceful stop).
kill -9 sends SIGKILL (forceful stop). Use only if normal termination fails.
5 What does df -h show?
Displays disk usage in human-readable format. Helps check free space on partitions.
6 What is the du command?
Shows directory or file sizes. du -sh gives a summary of folder size.
7 What does chmod do?
Changes file permissions (read, write, execute) for user, group, and others. Supports numeric (755) and symbolic modes.
8 How is find used?
Searches files by name, type, size, or date. Example: find /path -name "*.log".
9 What is tail -f used for?
Displays live updates of a file (streaming logs). Shows the last lines and updates as new lines are added.
10 What does crontab do?
Schedules recurring jobs. crontab -e opens the editor to add schedules.
11 What does scp do?
Securely copies files between remote systems over SSH (encrypted).
12 What is awk used for?
Text-processing tool for structured data. Works with patterns to extract fields (columns).
13 What is sed used for?
Stream Editor. Performs find/replace operations. Example: sed 's/old/new/g' file.txt.
14 What do who and whoami do?
who shows logged-in users. whoami shows the current user.
15 What is the top command?
Displays real-time CPU, memory, and process usage. Vital for performance debugging.
16 What does hostname do?
Shows the name of the current machine. Helps verify you are on the correct server.

Linux Concepts

Core Linux system concepts

1 What is Linux?
Open-source OS based on UNIX. Provides multitasking, multi-user support, and security. Used in servers, DevOps, and Cloud.
2 What are runlevels?
System states at boot. 0 (halt), 3 (multi-user), 5 (GUI), 6 (reboot). Controls which services start.
3 Hard Link vs Soft Link?
Hard Link: Points directly to inode. Cannot cross filesystems.
Soft Link: Shortcut pointing to a file path. Breaks if the target is deleted.
4 What is a zombie process?
A completed process that still has an entry in the process table because the parent didn't collect its exit status.
5 What is SSH?
Secure Shell. A protocol for secure remote login. Encrypts communication.
6 How to schedule jobs?
Use crontab -e for repeated jobs and at for one-time jobs. Cron format: Minute Hour Day Month Weekday.

Oracle SQL & PL/SQL

Database fundamentals and advanced concepts

1 What are constraints?
Rules to ensure data integrity. Types: PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, NOT NULL.
2 DELETE vs TRUNCATE vs DROP?
DELETE: Removes rows, can rollback.
TRUNCATE: Removes all rows instantly, cannot rollback.
DROP: Removes table structure.
3 Primary Key vs Foreign Key?
PK uniquely identifies a row (no NULLs). FK links to a PK in another table to ensure referential integrity.
4 What is a Stored Procedure?
Compiled code stored in DB. Accepts inputs, performs logic, returns outputs. Improves performance.
5 Function vs Procedure?
A Function must return a value and can be used in SELECT statements. Procedures may or may not return values.
6 What is a Trigger?
Code that executes automatically on events (INSERT, UPDATE, DELETE). Used for auditing.
7 RANK() vs DENSE_RANK()?
RANK(): Skips numbers on ties (1, 1, 3).
DENSE_RANK(): No skipping (1, 1, 2).
8 What is Normalization?
Organizing data to reduce redundancy by dividing large tables into smaller related ones.
9 What is an Index?
Structure to improve query speed. Too many indexes can slow down write operations (INSERT/UPDATE).
10 View vs Materialized View?
View: Virtual, fetches data on query.
Materialized View: Stores physical data, refreshes periodically.
11 What is NVL()?
Replaces NULL with a specific value. NVL(column, 0).
12 WHERE vs HAVING?
WHERE: Filters before grouping.
HAVING: Filters after GROUP BY.
13 UNION vs UNION ALL?
UNION: Removes duplicates (slower).
UNION ALL: Includes duplicates (faster).

Advanced Oracle Topics

Performance tuning and optimization

1 What are Hints in Oracle?
Hints provide instructions to the optimizer to choose a certain execution plan. Common hints: INDEX, PARALLEL, APPEND, NOCOPY, USE_HASH, USE_NL.
2 What is PARALLEL Hint?
Instructs the optimizer to use multiple server processes to execute SQL statements. Divides large tasks into smaller chunks and processes them using multiple CPU threads. Improves performance for large data volumes.
3 INDEX Hints
Instructs optimizer to use specific index instead of Full Table Scan.
SELECT /*+ INDEX(table_alias index_name) */ columns FROM table_name
4 APPEND Hints
Used with INSERT to enable direct-path insert, bypassing buffer cache and writing directly to datafiles.
INSERT /*+ APPEND */ INTO target_table SELECT * FROM source_table;
5 NOCOPY Hints
Used in PL/SQL to optimize IN OUT and OUT parameters. Uses pass-by-reference instead of pass-by-value for large data like collections, records, or LOBs.
6 USE_HASH Hint
Forces Oracle to use Hash Join between tables. Best when joining large tables.
SELECT /*+ USE_HASH(t2) */ ... FROM table1 t1 JOIN table2 t2
7 USE_NL Hint
Forces Oracle to use Nested Loop Join. Best when one table is small and indexes are available on join column.
8 Advantages of Triggers?
Triggers are PL/SQL subprograms associated with tables, schemas, or databases. They fire implicitly on events (INSERT, UPDATE, DELETE). Types: Statement Level and Row Level triggers.
9 DML Error Logging
Captures errors during DML operations without stopping the entire operation. Uses separate error logging table for constraint violations. Useful in bulk data loads with REJECT LIMIT.
10 Unique Index vs Unique Key
Unique Key Constraint enforces uniqueness without duplicate values (except NULL). Oracle creates unique index automatically when you create primary key or unique key.

AutoSys Advanced Topics

Q How many types of jobs are available in Autosys?
Main job types:
  • Command Job: Executes shell scripts, programs, or commands
  • Box Job: Container that groups related jobs together
  • File Watcher Job: Monitors file existence or modification
  • FTP Job: Transfers files between systems
Q Main difference between command jobs, box jobs, and file watcher jobs?
Command Job: Runs executable code (scripts/programs) on a specified machine.

Box Job: Logical container that doesn't execute code itself but controls the flow of jobs inside it. Used for grouping and dependency management.

File Watcher Job: Monitors a specific file path and triggers when the file appears or is modified. Doesn't execute commands but enables downstream jobs.
Q What types of job states are available in Autosys?
  • INACTIVE: Job is defined but not scheduled to run
  • ACTIVATED: Job is ready and waiting for start conditions
  • STARTING: Job is being initiated
  • RUNNING: Job is currently executing
  • SUCCESS: Job completed successfully
  • FAILURE: Job failed during execution
  • TERMINATED: Job was killed manually
  • ON_HOLD: Job is paused and won't run
  • ON_ICE: Job is bypassed but passes success to downstream
Q Difference between 'active' state and 'starting' state?
ACTIVATED: Job is enabled and waiting for its start conditions to be met (time, dependencies, etc.). It's ready but not yet initiated.

STARTING: All conditions are met and the job is in the process of being launched. The system is allocating resources and initiating execution.
Q If a box contains 10 jobs and nine succeed but one fails, what is the status of the box?
The box status will be FAILURE. A box job's status reflects the worst status of its child jobs. Even if only one job fails, the entire box is marked as FAILURE. This is important for dependency management and alerting.
Q What is a dependency in AutoSys?
A dependency is a condition that must be satisfied before a job can start. Types include:
  • Success Dependency: Job runs only if another job succeeds
  • Failure Dependency: Job runs only if another job fails
  • Done Dependency: Job runs when another job completes (success or failure)
  • Time Dependency: Job runs at a specific time
Defined using condition attribute in JIL.
Q File Watcher Job - Minimum requirements to run daily at 6:00 AM IST? 
insert_job: FILE_WATCH_JOB
job_type: FW
machine: server_name
watch_file: /path/to/file.txt
watch_file_min_size: 1
start_times: "06:00"
run_calendar: weekdays
owner: username@domain
permission: gx,ge,wx,we
alarm_if_fail: 1
Q Extra attributes for File Watcher job?
  • watch_interval: How often to check for the file (in seconds)
  • watch_file_min_size: Minimum file size to consider (prevents empty files)
  • watch_file_change_time: Monitors file modification time
  • max_run_alarm: Alert if file doesn't appear within expected time
Q Difference between auto profile variable and global variable?
Auto Profile Variable: Environment variables set automatically when a job runs on a specific machine. Defined in the agent's profile and loaded at job execution. Machine-specific.

Global Variable: Variables defined at the AutoSys instance level using JIL. Available to all jobs across all machines. Centrally managed and can be updated without changing individual job definitions.

Example: export GLOBAL_VAR=value vs insert_autosys_global: VAR_NAME

Multiple methods:

Q How to remove lines 30 to 50 from a text file?
Using sed:
sed '30,50d' filename.txt > newfile.txt

In-place editing:
sed -i '30,50d' filename.txt

Using awk:
awk 'NR<30 || NR>50' filename.txt > newfile.txt
Q Does sed delete lines permanently or temporarily?
Without -i flag: Changes are TEMPORARY. Output goes to stdout, original file unchanged.
sed '30,50d' file.txt (displays result, file unchanged)

With -i flag: Changes are PERMANENT. Modifies the original file.
sed -i '30,50d' file.txt (file is modified)

Best practice: Always create a backup before using -i:
sed -i.bak '30,50d' file.txt (creates file.txt.bak)
Q How to delete files older than 7 days?
Using find command:
find /path/to/directory -type f -mtime +7 -delete

With confirmation:
find /path/to/directory -type f -mtime +7 -exec rm {} \;

Delete specific file types:
find /path -name "*.log" -mtime +7 -delete

List before deleting (dry run):
find /path -type f -mtime +7 -ls

Q How to find the third-highest salary?
Method 1: Using DENSE_RANK() 
SELECT salary
FROM (
    SELECT salary, 
           DENSE_RANK() OVER (ORDER BY salary DESC) as rank
    FROM employees
)
WHERE rank = 3;
Method 2: Using subquery 
SELECT MAX(salary)
FROM employees
WHERE salary < (
    SELECT MAX(salary)
    FROM employees
    WHERE salary < (
        SELECT MAX(salary)
        FROM employees
    )
);
Q Difference between inner join and outer join?
INNER JOIN: Returns only matching rows from both tables. If no match exists, the row is excluded.
SELECT * FROM A INNER JOIN B ON A.id = B.id

OUTER JOIN: Returns matching rows PLUS unmatched rows from one or both tables (with NULLs for missing data).

Key difference: INNER JOIN is restrictive (intersection), OUTER JOIN is inclusive (union with NULLs).
Q Different types of outer joins?
LEFT OUTER JOIN (LEFT JOIN):
Returns all rows from left table + matching rows from right table. NULLs for non-matching right table rows.
SELECT * FROM A LEFT JOIN B ON A.id = B.id

RIGHT OUTER JOIN (RIGHT JOIN):
Returns all rows from right table + matching rows from left table. NULLs for non-matching left table rows.
SELECT * FROM A RIGHT JOIN B ON A.id = B.id

FULL OUTER JOIN:
Returns all rows from both tables. NULLs where no match exists.
SELECT * FROM A FULL OUTER JOIN B ON A.id = B.id
Q UNION vs UNION ALL - Which is faster?
UNION:
- Combines results and removes duplicates
- Performs sorting to identify duplicates
- SLOWER due to duplicate elimination

UNION ALL:
- Combines results and keeps all rows including duplicates
- No sorting or duplicate checking
- FASTER (recommended when duplicates are acceptable)

Performance tip: Use UNION ALL when you know there are no duplicates or when duplicates don't matter.
Q Difference between GROUP BY, HAVING, and ORDER BY?
GROUP BY: Groups rows with same values into summary rows. Used with aggregate functions (COUNT, SUM, AVG).
SELECT dept, COUNT(*) FROM emp GROUP BY dept

HAVING: Filters groups AFTER grouping. Works with aggregate functions.
SELECT dept, COUNT(*) FROM emp GROUP BY dept HAVING COUNT(*) > 5

ORDER BY: Sorts the final result set. Applied at the end.
SELECT * FROM emp ORDER BY salary DESC

Execution order: WHERE â†’ GROUP BY â†’ HAVING â†’ ORDER BY
Q What is an inline view?
An inline view is a subquery in the FROM clause that acts as a temporary table. Also called a "derived table".

Example: 
SELECT dept_name, avg_salary
FROM (
    SELECT department_id, 
           AVG(salary) as avg_salary
    FROM employees
    GROUP BY department_id
) emp_avg
JOIN departments d 
ON emp_avg.department_id = d.department_id
WHERE avg_salary > 50000;
Benefits: Simplifies complex queries, improves readability, enables multi-level aggregations.
Q Extract name from email using INSTR and SUBSTR
Problem: Extract "john" from "john@gmail.com"

Solution: 
SELECT 
    email,
    SUBSTR(email, 1, INSTR(email, '@') - 1) as username
FROM users;

-- Example:
-- email: john@gmail.com
-- INSTR(email, '@') returns 5
-- SUBSTR(email, 1, 4) returns 'john'
Explanation:
- INSTR(email, '@') finds position of @
- SUBSTR(email, 1, position-1) extracts from start to before @

âš™ï¸ Bulk Processing
BULK COLLECT: Fetch multiple rows in a single operation
DECLARE
    TYPE emp_tab IS TABLE OF employees%ROWTYPE;
    l_emps emp_tab;
BEGIN
    SELECT * BULK COLLECT INTO l_emps
    FROM employees
    WHERE department_id = 10;
END;
FORALL: Execute DML for multiple rows efficiently
FORALL i IN 1..l_emps.COUNT
    UPDATE employees
    SET salary = salary * 1.1
    WHERE employee_id = l_emps(i).employee_id;
Benefits: Reduces context switches, significantly improves performance for large datasets.

Table and Index Partitioning:

  • Improves query performance through partition pruning
  • Enhances maintenance operations (backup, archiving)
Partition Types:
  • Range Partitioning: Based on date ranges (e.g., monthly sales)
  • List Partitioning: Based on discrete values (e.g., regions)
  • Hash Partitioning: Evenly distributes data
Example:
CREATE TABLE sales (...) PARTITION BY RANGE (sale_date) (...)

Purpose: Precompute and store aggregated or complex query results

Benefits:
  • Reduces runtime calculations for complex queries
  • Improves performance for reporting and analytics
